Navigation

Four navigation patterns: the app header at the top, the sidebar nav on the left, breadcrumbs above the page title, and tabs for filtered views of the same page.

App header

One per page. Houses the brand, the primary nav, and the account menu.

Janeway Editorial
EditorialProductionReports
<header class="app-header">
  <a class="app-header-brand" href="/">
    <svg …></svg> Janeway
    <span class="app-header-chip">Editorial</span>
  </a>
  <nav class="app-header-nav" aria-label="Primary">
    <a href="#">Editorial</a>
    <a href="#">Production</a>
    <a href="#">Reports</a>
  </nav>
  <button type="button" class="app-header-avatar"
          aria-label="Account menu — signed in as t.kowalski">t.kowalski</button>
</header>

Tabs (filter)

Real navigation links — each tab is a different filtered view. Use aria-current="true" on the active link, not role="tab" (this is not an ARIA tabs widget).

  • Round 1 closed
  • Round 2 active
<nav class="tabs" aria-label="Review rounds">
  <ul>
    <li><a href="?round=1">Round 1 <span class="tabs-count">closed</span></a></li>
    <li><a href="?round=2" aria-current="true">Round 2 <span class="tabs-count">active</span></a></li>
  </ul>
</nav>
{% raw %}{% from "voyager/components/tabs/tabs.j2" import tabs, tab %}

{% call tabs("Review rounds") %}
  {{ tab("Round 1", url="?round=1", count="closed") }}
  {{ tab("Round 2", url="?round=2", count="active", current=True) }}
{% endcall %}{% endraw %}
