Function filter

Search input plus a radio-chip filter group. Used above a function-card grid to narrow visible cards. The chip active state uses :has(input:checked) — no JavaScript is required for the visual; only for actually filtering the grid.

Default

<form class="function-filter" role="search" aria-label="Filter functions">
  <div class="function-filter-search">
    <span class="function-filter-search-icon" aria-hidden="true">⌕</span>
    <label class="visually-hidden" for="fn-search">Search</label>
    <input type="search" name="q" id="fn-search" class="function-filter-input"
           placeholder="Search functions…" autocomplete="off">
  </div>
  <div class="function-filter-tags" role="radiogroup" aria-label="Filter by category">
    <label class="function-filter-tag"><input type="radio" name="tag" value="all" checked> All</label>
    <label class="function-filter-tag"><input type="radio" name="tag" value="settings"> Settings</label>
    …
  </div>
</form>

Empty state

Show when the filter returns zero results. Pair with JavaScript that toggles its hidden attribute.

No functions match your search.

  <span class="function-filter-empty-icon" aria-hidden="true">⌕</span>
  No functions match your search.
</div>

Client-side JS driver

Filtering itself runs in vanilla JS. The script reads data-title, data-desc, and data-tags from each function-card and toggles its hidden attribute.

(function () {
  var form = document.getElementById('manager-filter');
  var search = document.getElementById('mgr-search');
  var grid = document.getElementById('manager-grid');
  var empty = document.getElementById('manager-empty');

  function activeTag() {
    var c = form.querySelector('input[name="tag"]:checked');
    return c ? c.value : 'all';
  }

  function apply() {
    var q = (search.value || '').trim().toLowerCase();
    var tag = activeTag();
    var anyVisible = false;
    grid.querySelectorAll('.function-card').forEach(function (card) {
      var title = (card.dataset.title || '').toLowerCase();
      var desc  = (card.dataset.desc  || '').toLowerCase();
      var tags  = (card.dataset.tags  || '').toLowerCase().split(/\s+/);
      var matchTag  = tag === 'all' || tags.indexOf(tag) !== -1;
      var matchText = !q || title.indexOf(q) !== -1 || desc.indexOf(q) !== -1;
      card.hidden = !(matchTag && matchText);
      if (!card.hidden) anyVisible = true;
    });
    empty.hidden = anyVisible;
  }

  search.addEventListener('input', apply);
  form.addEventListener('change', apply);
})();

HTMX alternative

For server-side filtering, hand the same form to HTMX and target the grid container. The macro exposes optional action and htmx_target arguments.

{% raw %}{% call function_filter(
    action="{{ url('manager:dashboard') }}",
    htmx_target="#manager-grid"
) %}
  {{ function_filter_search(name="q", value=q) }}
  …
{% endcall %}{% endraw %}

Notes

  • The radio chips are real form inputs — the active state is driven entirely by CSS (:has(input:checked)). No JS needed for the visual.
  • Wrap the radio group in a div role="radiogroup" with aria-label so screen readers announce it as a group.
  • For accessibility, the visible label text is also the radio's value in the form submission — no visually-hidden labels needed.
  • Pair every filter row with an function-filter-empty block hidden by default; toggle via JS or server response.
