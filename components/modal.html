Modal

Dialog with a backdrop, header, body, and footer. Used for confirmations and short focused actions. Add .is-open to the backdrop to show it. JS handles ESC to close, focus trapping, and restoring focus on close.

Confirm a destructive action

The modal below is rendered inline as a static demo (no backdrop overlay).

Withdraw reviewer assignment

Withdrawing this reviewer will close their assignment and notify them by email. The author will not be informed.

You can replace the reviewer afterwards from the reviewer list.

<div class="modal-backdrop is-open" role="dialog" aria-modal="true" aria-labelledby="m-1">
  <div class="modal">
    <div class="modal-head">
      <h2 class="modal-title" id="m-1">Withdraw reviewer assignment</h2>
      <button class="modal-close" type="button" aria-label="Close dialog">×</button>
    </div>
    <div class="modal-body">
      <p>Withdrawing this reviewer will close their assignment …</p>
    </div>
    <div class="modal-foot">
      <button class="btn">Cancel</button>
      <button class="btn btn-warning">Withdraw reviewer</button>
    </div>
  </div>
</div>
{% raw %}{% from "voyager/components/modal/modal.j2" import modal, modal_body, modal_foot %}
{% from "voyager/components/button/button.j2" import button %}

{% call modal("Withdraw reviewer assignment", id="m-1", is_open=True) %}
  {% call modal_body() %}
    <p>Withdrawing this reviewer will close their assignment …</p>
  {% endcall %}
  {% call modal_foot() %}
    {{ button("Cancel") }}
    {{ button("Withdraw reviewer", kind="warning") }}
  {% endcall %}
{% endcall %}{% endraw %}

Notes

  • Always set aria-modal="true" and aria-labelledby on the dialog.
  • The opening control should receive focus back when the modal closes.
  • Reserve modals for short focused tasks. For longer flows (a decision form, a settings page), navigate to a full page.
  • Don't nest modals.
