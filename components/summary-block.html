Summary block

Bordered container with a grey header strip (title + actions) and a body that typically holds a summary list. Replaces ad-hoc card boxes across the system.

With summary list

The most common pairing. Each row is one fact: label, value, optional change link.

Nested rows with status rails

A summary-block can wrap a stack of child summary-blocks to present a grouped list under a single header — for example, the reviewers on a peer-review screen. Children sit edge-to-edge inside a summary-block-body-flush body; their outer borders are stripped automatically and a single hairline separates each row.

Add one of summary-block-rail-green, summary-block-rail-amber, or summary-block-rail-red to each child to surface its state as a 4px coloured rail on the left edge of the parent. This matches the rail idiom on article-card so editors learn the colours once.

Notes

  • The title is always an <h2> — summary blocks live under the page H1.
  • Set aria-labelledby on the section to the title's id so screen readers announce the block by name.
  • Use summary-block-body-flush when the body is a summary list — it removes the body padding so the list rows go edge-to-edge.
  • Rails are state, not decoration. Don't use them for visual variety; reserve them for telling the reader something they need to know at a glance.
