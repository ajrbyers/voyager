Form

Inspired by the UK Government Design System form patterns. Conventions Voyager picks up from GDS: mark optional fields (never required), error messages use <p> with a visually-hidden “Error:” prefix, error groups get a red left border, aria-describedby lists the hint first then the error, and radio/checkbox controls are 40 × 40 px.

Text input

Optional fields append “(optional)” to the label. Don't mark required fields — assume required by default.

As it should appear on the published article 
<div class="form-group">
  <label class="form-label" for="full-name">Full name (optional)</label>
  <span class="form-hint" id="full-name-hint">As it should appear on the published article</span>
  <input class="form-input" id="full-name" type="text" aria-describedby="full-name-hint">
</div>
{% raw %}{% from "voyager/components/form/form.j2" import form_text %}

{{ form_text(
    name="full_name", id="full-name",
    label="Full name", optional=True,
    hint="As it should appear on the published article"
) }}{% endraw %}

Width modifiers

Fixed character widths on inputs that should be sized to their content (postcodes, reference numbers, dates).

<input class="form-input form-input-width-10" type="text" autocomplete="postal-code">
{% raw %}{{ form_text(
    name="postcode", id="postcode",
    label="Postcode", width=10, autocomplete="postal-code"
) }}{% endraw %}

Available: form-input-width-2, -3, -4, -5, -10, -20. For fluid widths use form-width-full, form-width-three-quarters, form-width-two-thirds, form-width-one-half, form-width-one-third, form-width-one-quarter.

Textarea with character count

The character-count wrapper sits outside the form-group; the count message is a sibling div with an id referenced by the textarea's aria-describedby. Order in aria-describedby: count message id, then hint, then error.

You can enter up to 4,000 characters
<div class="form-character-count" data-maxlength="4000">
  <div class="form-group">
    <label class="form-label" for="letter">Decision letter</label>
    <textarea class="form-textarea" id="letter" rows="5"
              maxlength="4000"
              aria-describedby="letter-info"></textarea>
  </div>
  <div id="letter-info" class="form-character-count-message">
    You can enter up to 4,000 characters
  </div>
</div>

    name="letter", id="letter",

Date input

Three numeric text inputs (day, month, year) inside a fieldset with role="group". Day and month use form-input-width-2; year uses form-input-width-4. inputmode="numeric" brings up the numeric keypad on mobile.

Error state

Three things change in an error state:

  1. Add form-group-error to the wrapping form-group — gives the red left border + padding.
  2. Add the error message as a <p class="form-error-message"> with a visually-hidden “Error:” prefix span.
  3. Add form-input-error (or form-textarea-error / form-select-error) to the control. Set aria-describedby to include the error id. If a hint is also present, hint id comes first, then error id.

Error: Enter a date in the future

<div class="form-group form-group-error">
  <label class="form-label" for="due">Due date</label>
  <p class="form-error-message" id="due-error">
    <span class="visually-hidden">Error:</span> Enter a date in the future
  </p>
  <input class="form-input form-input-error" id="due" type="date"
         aria-describedby="due-error">
</div>
{% raw %}{{ form_text(
    name="due", id="due", label="Due date", type="date",
    error="Enter a date in the future"
) }}{% endraw %}

Password input

Password field with a Show / Hide toggle. The toggle is a sibling button; a small JS handler swaps the input's type attribute between password and text. aria-controls ties the button to the input it controls.

Use at least 12 characters, mixing letters, numbers and symbols 
<div class="form-group">
  <label class="form-label" for="password">Password</label>
  <span class="form-hint" id="password-hint">Use at least 12 characters…</span>
  <div class="form-password">
    <input type="password" id="password" class="form-input"
           aria-describedby="password-hint" autocomplete="new-password">
    <button type="button" class="form-password-toggle"
            aria-controls="password" data-form-password-toggle>Show</button>
  </div>
</div>
{% raw %}{% from "voyager/components/form/form.j2" import form_password %}

{{ form_password(
    name="password", id="password",
    label="Password",
    hint="Use at least 12 characters, mixing letters, numbers and symbols",
    autocomplete="new-password"
) }}{% endraw %}

Tiny driver — wire once at page level:

document.addEventListener('click', function (event) {
  var btn = event.target.closest('[data-form-password-toggle]');
  if (!btn) return;
  var id = btn.getAttribute('aria-controls');
  var input = document.getElementById(id);
  if (!input) return;
  var showing = input.type === 'text';
  input.type = showing ? 'password' : 'text';
  btn.textContent = showing ? 'Show' : 'Hide';
});

Heading-as-label / heading-as-legend

When a form has a single question, that question is the page heading. GDS recommends wrapping the label in an <h1> (single-input forms) or wrapping an <h1> inside the legend (multi-input). The visual heading and the form label are the same element — announced once.

It's on your National Insurance card, benefit letter, payslip or P60. 
<div class="form-group">
  <h1 class="form-label-wrapper">
    <label class="form-label form-label-l" for="nino">What is your National Insurance number?</label>
  </h1>
  <span class="form-hint" id="nino-hint">It's on your National Insurance card …</span>
  <input class="form-input form-input-width-10" id="nino" type="text" aria-describedby="nino-hint">
</div>

Notes

  • Pair errors with an error summary at the top of the form. Each summary entry is a link that focuses the field.
  • Don't use placeholder text as a substitute for a label or hint.
  • Don't disable submit buttons while errors exist — let the user submit and see the error summary.
  • Group questions logically; one question per page where possible (GDS “one thing per page” pattern).
