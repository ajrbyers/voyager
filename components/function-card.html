Function card

Bordered, clickable card for a single “function” in the manager — a thing you can configure or open. Carries a coloured icon, an optional count, a title, a description, category tags, and an arrow. Forms the building block of the Manager dashboard.

Default

Icon, count, title, description, tags, arrow. Hover lifts the card and shifts the arrow right.

Coming-soon variant

Use is-coming-soon for functions that are documented but not yet available. The card dims, becomes non-clickable, and shows a yellow “Coming soon” badge in place of the count.

Filtering

For client-side filtering with function-filter, set three data attributes on each card so the JS can match:

  • data-title — lowercase title for text search
  • data-desc — lowercase description tokens for text search
  • data-tags — space-separated category tags for tag filter
<a href="#" class="function-card"
   data-title="content pages"
   data-desc="static cms pages about contact policies"
   data-tags="content">
  …
</a>

Notes

  • The card is a real <a> — focusable, keyboard-accessible by default.
  • Choose icon colour by category, not aesthetic — users learn that "blue means settings" over time.
  • Count is optional. If unknown or not meaningful, leave it off; the layout collapses gracefully.
  • Keep descriptions to one line where possible (the card has min-height to keep rows aligned).
