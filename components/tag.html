Tag & chip

Tags are small uppercase status labels (inspired by the GOV.UK pattern). Code chips are inline monospaced labels for IDs, branch names, and SHAs.

Tag variants

Default (info) Approved Awaiting Declined Article Open 
<span class="tag">Default</span>
<span class="tag tag-green">Approved</span>
<span class="tag tag-orange">Awaiting</span>
<span class="tag tag-red">Declined</span>
<span class="tag tag-grey">Article</span>
<span class="tag tag-blue">Open</span>

Status helpers

Three utility classes for inline severity colour:

4 days overdue — apply when a date has passed.

Awaiting acknowledgement — apply for in-progress states.

Decision sent — apply for completed positive outcomes.

<span class="status-overdue">4 days overdue</span>
<span class="status-pending">Awaiting acknowledgement</span>
<span class="status-success">Decision sent</span>
{% raw %}{# Status helpers are utility classes, not macros — use the class directly. #}
<span class="status-overdue">{{ days_overdue }} days overdue</span>{% endraw %}

Notes

  • Tags use both colour and text to convey status — never rely on colour alone.
  • Use tag-grey for neutral category labels (Article, Round 2 of 2). Reserve coloured tags for status.
  • Don't put tags inside form labels or headings unless the visual treatment makes sense — they're inline text, not buttons.
