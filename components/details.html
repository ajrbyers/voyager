Details

Collapsible disclosure block, GDS-inspired. Wraps the native <details>/<summary> elements so it works without JavaScript. Use sparingly — hidden content can be missed. Good for secondary help text, advanced options, or long file lists.

Default (closed)

Help with reviewer recommendations

You can recommend accept, minor revisions, major revisions, or reject. Your recommendation is shared with the handling editor only — it does not replace their decision.

If you decline to make a recommendation, leave this field empty and submit comments only.

Notes

  • Use <details> — never roll your own collapsible. Keyboard and screen-reader support is free.
  • The summary should describe what's inside, not "Show more". A user should know whether to click without trying.
  • Don't hide critical content. If a user must see it to complete a task, don't put it inside details.
